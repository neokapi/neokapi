## 1. Self-improvement & process patterns to adopt

**The verification loop is the unit of unattended work.** Every format-maintenance session must end by executing spec.yaml assertions + parity + maturity guardrails and pasting output as evidence ‚Äî "Give Claude a check it can run‚Ä¶ It's the difference between a session you watch and one you walk away from"; gate escalation: prompt-ask ‚Üí `/goal` ‚Üí Stop hook ‚Üí adversarial verification subagent in fresh context (https://code.claude.com/docs/en/best-practices). Routines caveat applies to any cron-driven triage: "A green status‚Ä¶ does not mean the task in your prompt succeeded" ‚Äî read the transcript (https://code.claude.com/docs/en/routines).

**Eval gates with a documented lifecycle.** Eval-driven development: "build evals to define planned capabilities before agents can fulfill them"; 20‚Äì50 cases from real failures suffice early; capability evals graduate into regression suites that must stay ~100% (https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents). spec.yaml suites are the metric; the L0‚ÄìL4 ladder is literally the capability‚Üíregression graduation path. Grader hierarchy: deterministic/code graders first; LLM judges only for what code can't check, with structured rubrics and an "Unknown" out. neokapi's deterministic oracles (round-trip byte-equality, Okapi parity, external validators) are the strongest possible grounding ‚Äî Reflexion-style loops only work with an oracle outside the model (https://www.emergentmind.com/topics/reflective-llm-based-agent, https://arxiv.org/pdf/2509.02547).

**Runbooks as skills with progressive disclosure.** Lean SKILL.md (<500 lines) per workflow pointing at `references/` (loaded on demand) and `scripts/` (executed, not read); low-freedom wording for fragile steps, high-freedom heuristics for triage judgment; descriptions are the routing layer (https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices, https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills). Bundle repeated work: "If all 3 test cases resulted in the subagent writing a create_docx.py‚Ä¶ bundle that script" ‚Äî write it once into `scripts/`.

**Close the loop after every run (Claude A/B).** End each triage/remediation run with a self-reflection step that PRs the skill/runbook itself, citing observed agent behavior, not speculation; test revisions with paired with-old/with-new subagent runs graded on named assertions before adoption (skill-creator harness: `benchmark.json`, 60/40 train/test split for description tuning ‚Äî https://github.com/anthropics/skills/blob/main/skills/skill-creator/SKILL.md). Anthropic's tool-testing agent measured a 40% task-time reduction from feeding failing transcripts back and letting the model rewrite tool descriptions, verified on held-out sets (https://www.anthropic.com/engineering/writing-tools-for-agents, https://www.anthropic.com/engineering/multi-agent-research-system).

**Ledger = structured note-taking outside the context window.** Persist four channels per the Ralph-loop pattern: (1) git commits per remediation, (2) chronological progress log, (3) machine-readable per-format status (the /format-maturity dataset), (4) semantic learnings file ‚Äî brief, factual, pruned on a schedule, archived not deleted; prefer fresh-context restarts with structured handoffs over ever-growing sessions (https://addyosmani.com/blog/self-improving-agents/, https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents).

**Prompt/rubric versioning with regression gates.** Every prompt edit is a regression risk ‚Äî generic "improvements" are not monotonic (26/30‚Üí9/30 documented drop, https://arxiv.org/pdf/2601.22025); gate edits with a fixed ‚âà20‚Äì30-case suite per prompt surface, promptfoo-style old-vs-new diff (https://github.com/promptfoo/promptfoo). The optimization loop is GEPA-shaped: metric returns score *plus feedback text*; a reflection pass proposes rubric/prompt edits; keep a Pareto archive of candidates scored on a fixed held-out format set; promote only versions that dominate (https://arxiv.org/abs/2507.19457, https://dspy.ai/getting-started/gepa-optimization/). Counter prompt rot structurally: move constraints into validators/schemas, pin model versions in CI, and on model upgrades follow the vendor runbook ‚Äî inventory ‚Üí freeze prompts ‚Üí swap model ‚Üí run eval to isolate delta ‚Üí delete stale workarounds one at a time (https://github.com/openai/skills/blob/724cd511c96593f642bddf13187217aa155d2554/skills/.curated/openai-docs/references/upgrade-guide.md, https://knightli.com/en/2026/05/15/gpt-5-5-prompting-guide/).

**Criteria drift is expected.** Graders cannot fully specify criteria up front ‚Äî grading surfaces them (EvalGen, https://arxiv.org/abs/2404.12272). Make rubric edits a first-class, versioned output of every triage run; re-score affected formats after any rubric change. Where an LLM judge exists (divergence-faithfulness), validate via critique shadowing: ~30+ binary pass/fail gradings with written critiques to >90% agreement; golden agreement set re-run on judge-model change; recalibrate at >20‚Äì25% divergence (https://hamel.dev/blog/posts/llm-judge/, https://deepchecks.com/llm-judge-calibration-automated-issues/). Binary per-criterion checks everywhere; no 1‚Äì5 scales (https://hamel.dev/blog/posts/field-guide/).

**Living-runbook drift anchors.** Anchor runbook claims to file paths + symbols + git provenance and check deterministically in CI (AST fingerprints, exit 1 on drift) ‚Äî "build tools that yell at your agents"; LLM doc-repair only as the second layer (https://fiberplane.com/blog/drift-documentation-linter/). Accumulate reusable fix patterns Voyager-style: verified executable artifacts (test + fixture + snippet) indexed by one-line descriptions, then curated (https://arxiv.org/abs/2305.16291).

## 2. What mature format ecosystems do ‚Äî copy / avoid, per project

**Pandoc** ‚Äî copy: three test tiers (golden with `--accept`, unit, command tests named after issues ‚Äî `5474-tables.md` is the bug‚Üíregression flywheel); hard rule "If you are adding a new writer or reader, you must include tests"; plugin escape valve (custom Lua readers/writers; standalone-library-first for Djot/Typst); wild-corpus readiness bar (mdoc reader shipped after ~99.7% pass on 3,500 OpenBSD man pages) (https://pandoc.org/CONTRIBUTING.html, https://github.com/jgm/pandoc/discussions/11569, https://duckrabbit.tech/articles/mdoc.html). Also copy its honesty: declared fidelity anchor ("conversions from formats more expressive than pandoc's Markdown can be expected to be lossy") and user-selectable per-construct loss policy (`--track-changes accept|reject|all`) (https://pandoc.org/MANUAL.html). Avoid: reader/writer support for the same construct rotting asymmetrically ‚Äî issues #4301/#4303 prove read and write cells must be tracked separately (https://github.com/jgm/pandoc/issues/4301).

**Apache Tika** ‚Äî copy: 1TB/3M-file shared corpus harvested from govdocs1, Common Crawl, and bug trackers, hosted off-repo; release gating by *statistical* A/B comparison (exceptions by category, text-length deltas, MIME-distribution shifts, common-words quality scores) because byte-exact goldens don't scale to millions of wild files; detection as a maintained data artifact (`tika-mimetypes.xml` + classpath overlay) (https://cwiki.apache.org/confluence/display/tika/TikaEvalOnVM, https://cwiki.apache.org/confluence/display/TIKA/TikaEval, https://github.com/apache/tika/blob/main/tika-core/src/main/resources/org/apache/tika/mime/tika-mimetypes.xml). Also copy the security doctrine: fork-per-parse, 5-min default task timeout, restart on OOM/hang; failure taxonomy "catchable exceptions vs evil OOMs vs permahangs"; "avoid running Tika in the same process as anything that matters" (https://cwiki.apache.org/confluence/display/TIKA/The+Robustness+of+Apache+Tika). Avoid: insufficient DoS fixes ‚Äî CVE-2022-30126's first two fixes failed; confirm fixes by fuzzing (https://tika.apache.org/security.html).

**LibreOffice** ‚Äî copy: bug-tracker harvesting (`get-bugzilla-attachments-by-mimetype`, idempotent, per-filetype dirs) so every user-breaking file becomes permanent regression ammunition; round-trip fan-out (import ‚Üí export to every applicable format) with **external validators as a second oracle** (officeotron, ODF Validator ‚Äî "a validation error in the exported files will generate a test failure"); per-commit-range publication of results with triage links (https://github.com/LibreOffice/core/blob/master/bin/get-bugzilla-attachments-by-mimetype, https://mmohrhard.wordpress.com/2014/03/24/update-about-our-importexport-crash-testing/, https://dev-builds.libreoffice.org/crashtest/). Avoid: nothing structural ‚Äî but note full runs take ~3 days, so this tier is out-of-band, never PR CI.

**FFmpeg FATE** ‚Äî copy: split storage (small refs in-tree, big samples out-of-tree via rsync); append-only sample governance ("generally do not replace, remove or overwrite files" so historical checkouts stay testable; minimize before upload); `GEN=1` accept-mode; distributed volunteer-fleet reporting (https://ffmpeg.org/fate.html, https://github.com/FFmpeg/FFmpeg/blob/master/doc/fate.texi).

**ExifTool** ‚Äî copy: solo-maintainer sustainability via crowdsourced inbound samples ("send me an e-mail and I'll add your new discoveries"); fully self-contained golden suite with deliberately minimized/truncated test images shipped in the tarball; golden-testing both read and write paths (`testCompare()` + `writeCheck()`) (https://exiftool.org/, http://web.mit.edu/graphics/src/Image-ExifTool-6.99/t/TestLib.pm).

**tree-sitter** ‚Äî copy: corpus DSL with per-test attributes (`:skip`, `:error`, `:language`, `:platform`); delimiter escaping by lengthening; and the crucial guard rail ‚Äî `-u` auto-update **refuses to rewrite tests containing ERROR/MISSING nodes**, so error-expectation tests can't be silently clobbered by accept-mode (https://tree-sitter.github.io/tree-sitter/creating-parsers/5-writing-tests.html, https://github.com/tree-sitter/tree-sitter/blob/master/docs/src/creating-parsers/5-writing-tests.md).

**unified/remark/micromark** ‚Äî copy: spec-compliant minimal core validated against the external spec's corpus; every dialect is a versioned extension package, containing drift outside core; explicit governance/funding for hundreds of packages (https://github.com/micromark/micromark, https://dev.to/murderlon/crowdsourcing-the-evolution-of-text-parsing-with-unified-5ao1).

**Okapi itself** ‚Äî avoid: hand-written, unversioned, untested per-filter limitation bullets ("Skeleton not supported," "Comments are lost‚Ä¶") that aren't enumerable across filters (https://okapiframework.org/wiki/index.php/XLIFF-2_Filter). This is the "vibes" state vocabulary.yaml replaces. Also avoid XLIFF's Statement-of-Use model: five voluntary self-declarations, no shared harness, unverifiable claims (https://archive-ouverte.unige.ch/unige:105631).

Cross-project invariants: two-tier corpora (small committed goldens + huge out-of-repo wild corpus); three oracles in combination (byte/hash-exact diff, external validators on written output, statistical A/B at scale); bug‚Üícorpus flywheel; accept-mode with guard rails; acceptance bar = maintainer burden, relieved by a plugin tier; upstream change-tracking is mostly inbound samples, not polling.

## 3. Executable-spec design (spec.yaml schema features)

The recurring case grammar across CommonMark, JSON-Schema-Test-Suite, toml-test, yaml-test-suite, WPT, test262, Unicode DDT, and XLIFF test-suite (https://github.com/commonmark/commonmark-spec, https://github.com/json-schema-org/JSON-Schema-Test-Suite, https://github.com/toml-lang/toml-test, https://github.com/yaml/yaml-test-suite, https://web-platform-tests.org/test-suite-design.html, https://github.com/tc39/test262/blob/main/CONTRIBUTING.md, https://github.com/unicode-org/conformance, https://github.com/oasis-tcs/xliff-xliff-22):

1. **Case shape**: `{stable-id (4‚Äì6 char, never reused), name, input (inline|file-ref), class: valid|invalid|operation, expected (one or more views), cite, tags, since/until version bounds, origin}`. One fault per invalid case, named after the fault (toml-test; XLIFF `bad_DataIdNotUnique.xlf`).
2. **Three validity classes**: `valid/` (parse + extraction assertions), `invalid/` (must reject, with error-category assertion), `in-out/` operation pairs (extract‚Üíblocks, merge(blocks‚Ä≤)‚Üínative, segment, redact) ‚Äî XLIFF 2.1's `core/in-out/` transformation pairs are the closest existing analog to neokapi's read‚Üítransform‚Üíwrite unit.
3. **Neutral expected-encoding**: never express expected output in the format under test ("a data exchange format is needed‚Ä¶ TOML cannot be used, as it would imply that a particular parser has a blessing of correctness" ‚Äî toml-test). Define a canonical JSON Part/block event-stream dump emitted deterministically by kapi; this prevents the Go engine being the blessed oracle, lets okapi-bridge/WASM/ports run the corpus via a thin stdin/stdout shim (DDT `#TEST` protocol), and gives AI a schema-checkable generation target.
4. **Multiple expected views per input** (yaml-test-suite's `test.event`/`in.json`/`out.yaml`): `expected.blocks` (event stream), `expected.roundtrip` (byte-exact or normalized native), `expected.extracted` (text-only), optional `expected.valid_by: <external-validator>`. Views map onto maturity levels (L1 = blocks view; L3 = roundtrip; L4 = all views + invalid corpus + validator).
5. **Machine-checkable spec citation**: `cite: {spec: <catalog-id>, version, url (with fragment), clause, heading, quote (‚â§1 normative sentence/~50 words), quote_sha256}` ‚Äî test262 `esid` + WPT `<link rel=help>` + BCD's "Each URL must contain a fragment identifier", lint-enforced; resolve against the **pinned snapshot**, not the network (see ¬ß5 knowledge base) (https://github.com/mdn/browser-compat-data/blob/main/schemas/compat-data-schema.md, https://web-platform-tests.org/writing-tests/lint-tool.html).
6. **Expectations live outside the corpus** (WPT's decisive pattern): a per-implementation `expectations/` tree (`<format>/<case-id>: FAIL (reason, issue#)`); expected-FAIL cases still run and report; `kapi spec-test --update-expectations` regenerates from run logs (https://web-platform-tests.org/tools/wptrunner/docs/expectation.html).
7. **Result taxonomy** (Unicode DDT): Success / Failure / Error / **Unsupported** / **Known-divergence** (documented, with citation for why neokapi intentionally differs) ‚Äî matches the existing faithful% framing (https://github.com/unicode-org/conformance).
8. **Meta-schema the corpus and gate in CI** (JSON-Schema `test-schema.json`; DDT's three checkpoints: after generation, before run, after run) ‚Äî the precondition for trusting AI-generated cases.
9. **Snapshot-update with guard rails**: `kapi spec-test -u <format>` rewrites expected views with git diff as review surface; **never auto-update invalid/malformed/error cases** (tree-sitter's ERROR-node exception).
10. **Versioning**: per-format-version manifests listing case IDs (toml-test `files-toml-1.0.0`) rather than duplicated cases; immutable dated releases if shared externally (yaml `data-YYYY-MM-DD`); `origin: {suite, version, upstream-id}` so wholesale imports (CommonMark `--dump-tests`, toml-test, yaml-test-suite, JSON Schema suite, XLIFF test-suite, UCD fixtures) stay idempotent.
11. **AI generation loop** (grounded in iPanda/DiffSpec, https://arxiv.org/pdf/2507.00378, https://arxiv.org/pdf/2410.04249, https://arxiv.org/html/2510.23350v1): per spec-section anchor ‚Üí retrieve clause text ‚Üí generate positive *and* negative candidates ‚Üí schema-validate ‚Üí execute against neokapi *and* okapi-bridge as differential oracle ‚Üí auto-classify (both-agree = candidate-pass; disagree = divergence triage; both-reject = candidate-invalid) ‚Üí human reviews only diffs. Per-section case counts surface spec-coverage gaps (CommonMark `-s`).
12. **Companion `vocabulary.yaml` per format** (followup-representation-fidelity): construct registry seeded from ITS 2.0's 19 data categories + XLIFF 2.x modules + XLIFF inline-code vocabulary (https://www.w3.org/TR/its20/, https://docs.oasis-open.org/xliff/xliff-core/v2.2/xliff-extended-v2.2-part2.html); per construct: `realization`, `spec` cite, `expressible` (TTML content-profile dimension, https://www.w3.org/TR/ttml2/), **separate `read`/`write` cells** (pandoc asymmetry lesson), `status: lossless|lossy|dropped|rejected|unknown` (caniuse `y/a/n/u` + Contentful reject), mandatory `notes` whenever not lossless (BCD `partial_implementation` rule), `policy: preserve-as-skeleton|preserve-as-overlay|drop|error`, and `evidence:` pointing at passing spec.yaml case IDs ‚Äî claims-must-bind-to-tests, WPT `WEB_FEATURES.yml` style (https://github.com/web-platform-tests/wpt/blob/master/css/css-transforms/WEB_FEATURES.yml). Fidelity computation: score only expressible constructs; partial never counts as support (Baseline rule, https://github.com/web-platform-dx/web-features/blob/main/docs/baseline.md). Three nested round-trip tiers as evidence vocabulary: text-content equality ‚äÇ canonical construct-dump equality (ITS test-suite normalization trick, https://github.com/w3c/its-2.0-testsuite) ‚äÇ byte equality (SCORE-Bench's "distinguish legitimate representational variation from actual data loss", https://unstructured.io/blog/introducing-score-bench-an-open-benchmark-for-document-parsing).

## 4. Reference-corpus infrastructure recipe

**Three tiers per format** (reference-corpora.md ¬ßimplications, validated by every ecosystem in ¬ß2):
- **Tier A "exemplars" (in-repo)**: ‚â§ a few MB/format; only CC0/self-created/US-gov files (OPF format-corpus is all-CC0 by construction, https://github.com/openpreserve/format-corpus); REUSE/SPDX sidecar per binary fixture (`<file>.license` or `REUSE.toml`; `reuse lint` in CI so an unlicensed fixture can't land, https://reuse.software/spec-3.3/). Runs in PR CI. Cap size; each file must justify itself via a distinct assertion or coverage delta (corpus-distillation result: minimized beats huge, https://dl.acm.org/doi/pdf/10.1145/3460319.3464795).
- **Tier B "harvested" (fetch-on-demand, never vendored)**: govdocs1 by-type tarballs (US-gov, redistributable, https://digitalcorpora.org/corpora/file-corpora/files/), NapierOne (44 types √ó ~5k, attribution license, https://github.com/simonrdavies/NapierOne), SafeDocs PDF shards (https://digitalcorpora.org/corpora/file-corpora/cc-main-2021-31-pdf-untruncated/), UNSAFE-DOCS as the hostile set (https://digitalcorpora.org/corpora/file-corpora/unsafe-docs-cc-main-2021-31-unsafe/), plus bug-tracker attachments. Cached by sha256 in XDG cache via `make corpus FORMAT=x`. Common Crawl rule: CC grants no license to crawled content ‚Äî fetch-on-demand only, `redistributable: false` (https://commoncrawl.org/terms-of-use). When harvesting CC, refetch truncated payloads (1MB cap ‚Äî the SafeDocs lesson, https://github.com/tballison/CC-MAIN-2021-31-PDF-UNTRUNCATED). Nightly/weekly sweeps, never PR CI.
- **Tier C "synthetic" (generated, deterministic)**: seeded Go generators per format (LLM writes the generator, not the files ‚Äî reviewable, reproducible, https://arxiv.org/pdf/2501.19282); Go fuzz corpora in `testdata/fuzz/<FuzzName>/` (https://go.dev/doc/security/fuzz/); skeleton-suite-style minimal magic shells from PRONOM signatures for detection tests (https://www.digipres.org/formats/); producer-variance matrix for office formats (same document saved by Word/LibreOffice/Google Docs/python-docx ‚Äî self-created ‚Üí CC0 ‚Üí Tier A, OPF office-examples pattern, https://github.com/openpreserve/format-corpus/blob/master/office-examples/README.md); FormatFuzzer 010-templates or libprotobuf-mutator IR for binary containers (https://arxiv.org/html/2109.11277, https://github.com/google/libprotobuf-mutator). Fuzzing axes for fixture generation: construct √ó document-shape √ó producer-tool; generate inputs, never expected outputs (https://hamel.dev/blog/posts/field-guide/).

**Provenance manifest (`corpus.yaml` per format, sibling to spec.yaml)** ‚Äî fields = OPF metadata template made machine-readable + REUSE/SPDX + content addressing (https://github.com/openpreserve/format-corpus/blob/master/metadata-template.ext.md): `sha256, size, format+version, puid (PRONOM), loc_fdd?, mime, ext_declared vs type_detected, creator_tool(+url), spec_url, origin: vendored|url|archive-member|bug|generated, source_url (+archive/member for zip shards), harvest_date, license (SPDX; LicenseRef-Unverified only in Tier B), license_evidence, redistributable: bool, defects: [], notes`.

**Licensing red lines**: vendor only CC0/PD/US-gov/own-created; NapierOne/ODC-By vendorable with recorded attribution; CC/web-harvested/bug attachments stay fetch-on-demand unless cleared; OSS-Fuzz's rule "ensure your seed corpus has an appropriate and consistent license, and do not include proprietary or sensitive data" (https://github.com/google/oss-fuzz/blob/master/docs/faq.md).

**Corpus invariants in CI**: run Siegfried (Go, embeddable, PRONOM+freedesktop signatures, https://github.com/richardlehane/siegfried) over every corpus file and assert detected PUID == manifest `puid` ‚Äî simultaneously regression-tests neokapi's own detection against registry ground truth and catches corrupted/mislabeled entries.

**Bug‚Üícorpus flywheel, institutionalized**: every format bug fix lands (a) a minimized failing document with `origin: bug, source_url: <issue>` and (b) a spec.yaml case named with the issue number (pandoc pattern); a GitHub-issue attachment harvester (LibreOffice's script, adapted) keeps it self-replenishing. Crashers from fuzzing/sweeps auto-promote to committed regression seeds (Go native fuzzing does this by design).

**Storage**: in-repo for Tier A; GitHub release assets for the frozen curated snapshot (2GiB/file, no bandwidth caps ‚Äî the `format-corpus` release + `fetch-corpus.sh`/`publish-corpus.sh` fetch-on-demand pattern, https://docs.github.com/en/repositories/releasing-projects-on-github/about-releases); HF dataset repo or a dedicated `neokapi/format-corpus` repo (clamav-fuzz-corpus precedent, https://github.com/Cisco-Talos/clamav-fuzz-corpus) if published; **not Git LFS** (metered bandwidth on every clone/fork, https://testdouble.com/insights/why-use-git-large-file-storage). FATE governance applies: append-only, minimize before adding.

**Sweep operations (security follow-up)**: one file = one subprocess (Tika ForkParser doctrine); ephemeral network-disabled runners only, never persistent self-hosted (https://www.sysdig.com/blog/how-threat-actors-are-using-self-hosted-github-actions-runners-as-backdoors); per-file wall-clock + RSS caps; classification taxonomy `OK / OK_ROUNDTRIP / EXPECTED_REJECT / CRASH / HANG / OOM / ROUNDTRIP_DRIFT`, with every CRASH/HANG/OOM/DRIFT minimized and promoted to `testdata/fuzz/`. Shared `core/safeio` primitives for all readers: byte budget (identical across CLI/server/WASM ‚Äî the file-type CVE-2026-32630 lesson, https://github.com/advisories/GHSA-j47w-4g3g-c36v), depth guard returning error not panic (go-yaml PR#515 precedent, https://github.com/go-yaml/yaml/pull/515), per-entry zip ratio+size caps (POI ZipSecureFile semantics: MinInflateRatio 1%, MaxEntrySize, https://poi.apache.org/apidocs/dev/org/apache/poi/openxml4j/util/ZipSecureFile.html), `filepath.IsLocal`/`os.Root` for any content-derived path (https://go.dev/blog/osroot), explicit decoder memory caps. cgo/bridge formats (okapi-bridge JVM, ICU, onnxruntime) re-introduce Rule-of-2 leg (b) and get subprocess isolation + timeout + memory cap + no network (https://chromium.googlesource.com/chromium/src/+/master/docs/security/rule-of-2.md, ExifTool CVE-2021-22204 https://www.sentinelone.com/vulnerability-database/cve-2021-22204/). Fuzz round-trip (read‚Üíwrite‚Üíread: no crash, idempotent, bounded resources) ‚Äî this also catches the Go XML round-trip-mutation class (CVE-2020-29509..11, unfixed in stdlib, https://mattermost.com/blog/coordinated-disclosure-go-xml-vulnerabilities/), which attacks neokapi's core value proposition. ClusterFuzzLite PR+batch mode now; OSS-Fuzz as stretch (https://google.github.io/clusterfuzzlite/, https://google.github.io/oss-fuzz/getting-started/accepting-new-projects/).

**Spec knowledge base alongside the corpus** (followup-spec-knowledge-base): `specs/catalog.yaml` (browser-specs pattern, https://github.com/w3c/browser-specs) + `snapshots/<spec>/<version>/` (vendor-class committed; cache-class gitignored) + `sections/<spec>/<version>/<anchor>.md` retrieval units (one numbered clause per file ‚Äî section-anchored beats embedding chunks on standards text, https://arxiv.org/pdf/2404.15939). Rights matrix: IETF/WHATWG/Ecma/Unicode-data/Google = vendorable; W3C-TR/OASIS/MS-* = unmodified copies + implementation excerpts with notice; Unicode prose/Apple = cache-or-link only; **ISO/IEC never** (single-user license ‚Äî use the free Ecma-376/OASIS-ODF twins, https://www.iso.org/files/live/sites/isoorg/files/store/en/PUB100206.pdf, https://ecma-international.org/policies/by-ipr/ecma-text-copyright-policy/). Citation lint resolves anchors against the pinned snapshot; a scheduled watch job diffs live `ids`/headings indexes (webref raw‚Üípatch‚Üícurated pipeline, 6-hour crawls, https://github.com/w3c/webref) and on breakage proposes `{new anchor, new clause}` via quote-hash search as an auto-PR.

## 5. Spec-watch + trend process

**Two watch tiers**: (a) spec drift on adopted formats; (b) emergence of new formats. Mechanism: pin `spec: {id, version, url, watch: [feeds]}` in each format's spec.yaml; a periodic skill fetches feeds, compares pinned versions, opens one issue per drift. Mechanics that exist today with zero auth: GitHub `releases.atom`/`commits.atom`, W3C news feed, Unicode blog Atom, OASIS announcements, Microsoft [MS-*] per-doc revision history + RSS (format-watch-trends.md ¬ßB7).

**The concrete watchlist** (full table in format-watch-trends.md ¬ßWatchlist; load-bearing entries):

## 6. Editor-embedding maturity ladder (orthogonal E-axis)

Five rungs, each gated by a documented API capability (editor-embedding.md ¬ß10) ‚Äî record `embedding: E0‚ÄìE4` + gate evidence per format in the maturity data:

- **E0 none** ‚Äî files only.
- **E1 faithful preview** ‚Äî any read path (REST export, `getFileAsync`, screenshots). All ecosystems permit.
- **E2 round-trip workflow** ‚Äî push/pull with **stable identity binding** (Figma layer‚Üîkey; Contentful entry‚Üîjob; IDML story IDs; bilingual docx). Canva fails the stable-identity test (random text order, no element identity, https://www.canva.dev/docs/apps/design-guidelines/content-querying-api/); browser overlays fail without pseudo-locale cooperation (https://support.crowdin.com/developer/in-context-localization/).
- **E3 embedded add-in (live panel)** ‚Äî reads the open document, writes targets in the editor's native idiom. Permitted: **Word (rich ‚Äî tracked changes + comments via WordApi 1.4**, https://learn.microsoft.com/en-us/javascript/api/requirement-sets/word/word-api-1-4-requirement-set), Figma (`setRange*` + font loading, https://developers.figma.com/docs/plugins/working-with-text/), InDesign UXP (direct DOM, `.ccx` direct distribution, https://developer.adobe.com/indesign/uxp/), Gutenberg (SlotFill + wp.data, https://developer.wordpress.org/block-editor/reference-guides/slotfills/), Sanity/Storyblok/Contentful, VS Code. **PowerPoint only partially** (text APIs ‚â•1.6, Dec 2024+; perpetual-license frozen at 1.5, iPad at 1.1 ‚Äî https://learn.microsoft.com/en-us/javascript/api/requirement-sets/powerpoint/powerpoint-api-requirement-sets). Google's modern add-on track (card UI only, no document context) and Canva structurally cap below E3 (https://developers.google.com/apps-script/add-ons/guides/workspace-restrictions).
- **E4 continuous bidirectional sync** ‚Äî editor events drive the pipeline. First-class only in CMS frameworks (Contentful app identities + event subscriptions, https://www.contentful.com/developers/docs/extensibility/app-framework/) and WordPress hooks; document editors have no content change feed (E4 there = polling exports).

Strategic readings: **the unoccupied E3 slots are Word and Google Docs** ‚Äî major TMSes treat .docx as a connector file format; Figma/CMS slots are saturated (Crowdin/Phrase/Smartling/Lokalise key-binding plugins are table stakes there, https://store.crowdin.com/figma). Exploit the Office asymmetry: `getFileAsync` extracts the full live .docx ‚Üí run the real openxml reader (server or in-pane WASM) ‚Üí write back via the object model; never reimplement extraction against office.js trees (https://learn.microsoft.com/en-us/office/dev/add-ins/develop/get-the-whole-document-from-an-add-in-for-powerpoint-or-word). The WASM cgo-less kapi build is the embedding payload everywhere except Figma's logic sandbox and Apps Script. Stable anchoring is the E2/E3 moat: define a canonical `editor-anchor` overlay (Word content-control tag / Figma node-id+range / Docs named-range / CMS field path) and let spec.yaml assert anchor survivability. Distribution shapes architecture: Office re-reviews only manifests (code redeploys free); Figma re-reviews every Community publish ‚Äî keep add-ins as stable shells over server/WASM behavior.

## 7. Top 10 design decisions

**D1 ‚Äî Split the promise from the score.** Publish two artifacts per format: a 3-level support tier (Supported / Maintained / Available) with release-gating semantics, changed only by explicit promotion/demotion events; and the L0‚ÄìL4+ maturity vector, recomputed every audit. Tier wording = what CI enforces ("regressions in this format's corpus, malformed suite, and Okapi parity block any release"), Rust-style ‚Äî a tier claim not backed by a CI gate is marketing (https://doc.rust-lang.org/rustc/target-tier-policy.html). K8s perma-beta is the documented failure mode of conflating them; import its fixes: `target_tier`+`target_by` per format, overdue = re-commit or park (https://github.com/kubernetes/enhancements/tree/master/keps/sig-architecture/1635-prevent-permabeta).

**D2 ‚Äî Headline by minimum, never average.** Publish the per-axis vector (engine fidelity, vocabulary fidelity, security/hardening, corpus, embedding); derive the headline tier as min over the designated gating axes, Baseline/Rust style. Scorecard's weighted aggregate is the cautionary tale ‚Äî the scalar is where measurement validity collapses (https://github.com/web-platform-dx/web-features/blob/main/docs/baseline.md, https://arxiv.org/pdf/2210.14884). Non-gating axes (embedding, knowledge assets) display as columns that inform but don't cap.

**D3 ‚Äî spec.yaml adopts the convergent case grammar with a neutral oracle encoding.** Stable IDs, valid/invalid/operation classes, multi-view expecteds, fault-named invalid cases, per-version manifests, `origin:` for imported suites, meta-schema gated in CI, and a canonical JSON block-event dump as the expected encoding so no implementation is blessed and any port/bridge/WASM runs the corpus via a ~30-line shim (https://github.com/toml-lang/toml-test, https://github.com/yaml/yaml-test-suite, https://github.com/unicode-org/conformance). Accept-mode (`-u`) with the tree-sitter guard rail: error/malformed expectations are never auto-updated.

**D4 ‚Äî vocabulary.yaml makes lossiness measurable and honest.** Repo-level construct registry seeded from ITS 2.0 categories + XLIFF 2.x modules + inline-code vocabulary; per-format expressible/supported matrix with separate read/write cells, status enum including `unknown`, mandatory notes on partial, policy enum for non-lossless handling, and `evidence:` binding every claim to passing spec.yaml cases with a CI drift gate. This replaces Okapi-wiki prose and the XLIFF SOU anti-pattern with the BCD/caniuse/WPT discipline (https://www.w3.org/TR/its20/, https://github.com/mdn/browser-compat-data/blob/main/schemas/compat-data-schema.md). Declare the pandoc-style fidelity anchor once: Block/Run/Overlay is the IR; loss is directional and enumerated.

**D5 ‚Äî Three-tier corpus with manifest-enforced provenance and licensing.** Tier A vendored CC0/own-created exemplars with REUSE sidecars (PR CI); Tier B fetch-on-demand harvested wild + hostile sets, sha256-cached, never vendored when license unclear (nightly/monthly); Tier C deterministic generators + fuzz seeds (regenerate, don't store). `corpus.yaml` carries the OPF-derived field set; Siegfried PUID verification is a corpus invariant; the bug‚Üícorpus flywheel is mandatory in every format bug fix; storage = release assets / dedicated repo, append-only, never LFS (https://github.com/openpreserve/format-corpus, https://reuse.software/spec-3.3/, https://github.com/richardlehane/siegfried, https://ffmpeg.org/fate.html).

**D6 ‚Äî Dual+statistical oracles above byte-equality.** For every format with an external validator (ODF Validator, OpenXML SDK/openxml-audit, Lynx/Schematron for XLIFF, `msgfmt -c`, CommonMark spec_tests.py), validate *neokapi-written* output across the corpus ‚Äî "actually legal per spec" is orthogonal to "same as Okapi" (https://odftoolkit.org/conformance/ODFValidator.html, https://github.com/dotnet/Open-XML-SDK). For wild-corpus scale, build a tika-eval-style A/B profiler (exception census by category, block/word-count deltas, roundtrip rate, detection-distribution shifts) gating releases on "no unexplained deltas," published per commit-range LibreOffice-style (https://cwiki.apache.org/confluence/display/TIKA/TikaEval, https://dev-builds.libreoffice.org/crashtest/).

**D7 ‚Äî Security/hardening as an explicit S0‚ÄìS4 axis.** S1 bounded (shared `core/safeio` budgets, identical limits across CLI/server/WASM; wild corpus 0 crash/hang/OOM), S2 fuzzed (committed Go fuzz targets read + round-trip, ClusterFuzzLite PR mode ‚Äî Scorecard `fuzzedWithGoNative` precedent), S3 hostile-hardened (UNSAFE corpus clean within fixed RSS/wall-clock; govulncheck clean; zero round-trip drift), S4 continuously assured (batch fuzzing, every crasher a committed seed, ‚â•30-day green sweeps). Tika's fork-isolation doctrine for everything cgo/bridge. The Tika taxonomy (catchable / evil-OOM / permahang, plus ROUNDTRIP_DRIFT) is the sweep classification (https://github.com/ossf/scorecard/blob/main/docs/checks.md, https://cwiki.apache.org/confluence/display/TIKA/The+Robustness+of+Apache+Tika, https://go.dev/doc/security/fuzz/).

**D8 ‚Äî Scorer/worker separation + default-to-decay certification.** The audit script, rubric, and assertion semantics live behind a change-control boundary: a remediation agent may not modify the scorer or relax an assertion in the same change that improves a score (CI path check); top tiers anchor on oracles the agent can't edit (Okapi parity, external corpora, resolvable citations); every "met" claim points at a checked-in evidence artifact (`{Met URL}` discipline); humans approve tier changes, agents do everything else (Rust recusal + TC39 two-implementations + Scorecard central recomputation, https://doc.rust-lang.org/rustc/target-tier-policy.html, https://tc39.es/process-document/). Every published tier carries `last_certified` + audit link; stale certification auto-displays and then demotes after grace ‚Äî a dead cron can't preserve a stale promise (CNCF's 2-month rule inverted, https://github.com/cncf/toc/blob/main/process/sandbox-annual-review.md). Demotion is normal: never delete, drop to a lower tier/plugin tier, announce before the release, keep a `demotions.md` ledger (https://github.com/tc39/proposals/blob/main/inactive-proposals.md). Version the rubric with stable criterion IDs; re-score fleet-wide on rubric change.

**D9 ‚Äî Spec knowledge base: pin or don't cite.** `catalog.yaml` + immutable snapshots + per-clause section files; citations are `{spec-id, version, url#fragment, clause, heading, quote ‚â§1 sentence, quote_sha256}` resolved against the pinned snapshot in CI; a non-blocking watch job diffs live anchor indexes and auto-PRs relocation proposals (webref/Bikeshed/WPT-lint mechanics, https://github.com/w3c/webref, https://speced.github.io/bikeshed/). Rights red lines: ISO text never enters repo/cache/prompts (use Ecma/OASIS twins); Apple link-only; CC-BY-SA files separately licensed; W3C/OASIS excerpts ride the implementation carve-outs (https://www.w3.org/copyright/document-license-2023/, https://www.iso.org/copyright.html). The section-anchored knowledge base is also the RAG unit for AI test generation (iPanda result: anchored retrieval + machine-validated cases + differential oracle + divergence taxonomy, https://arxiv.org/pdf/2507.00378).

**D10 ‚Äî The maintenance loop is a self-improving, eval-gated agent process on a calendar.** Weekly format-drift routine (merged PRs + watch feeds + sweep deltas ‚Üí issues), quarterly emergence scan, annual Oct‚ÄìNov Unicode-train upgrade window, yearly Office-fixture refresh. Skills are versioned, eval-gated artifacts: snapshot before revising, paired-run benchmark, promote only on held-out wins; every run writes its ledger entry and may PR the runbook citing observed behavior; rubric edits are versioned outputs of triage runs; model upgrades follow inventory‚Üífreeze‚Üíswap‚Üíisolate-delta‚Üítrim. The deterministic oracles (spec.yaml, parity, validators, corpus invariants) are the ground truth that makes all of this safe to run unattended (https://code.claude.com/docs/en/best-practices, https://code.claude.com/docs/en/routines, https://arxiv.org/abs/2507.19457, https://platform.claude.com/docs/en/agents-and-tools/agent-skills/enterprise).
