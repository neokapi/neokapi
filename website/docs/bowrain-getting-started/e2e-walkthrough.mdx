# End-to-End Walkthrough

This guide demonstrates a complete developer-to-translator localization journey
using the gokapi toolchain: from initializing a project in the CLI, to pushing
content to a Bowrain Server, inviting a translator, and pulling finished
translations back into the codebase.

We use a real Docusaurus site (included in the repo at `examples/docusaurus-e2e/`)
as the source project. The walkthrough has seven steps:

1. [Initialize the project](#step-1-initialize-the-project)
2. [Pseudo-translate for testing](#step-2-pseudo-translate-for-testing)
3. [Push to Bowrain](#step-3-push-to-bowrain)
4. [Claim the project](#step-4-claim-the-project)
5. [Invite a translator](#step-5-invite-a-translator)
6. [Translate in Bowrain](#step-6-translate-in-bowrain)
7. [Pull translations](#step-7-pull-translations)

---

## Prerequisites

- **Brain CLI** installed ([Installation](/docs/bowrain/installation))
- **Bowrain Server** running locally or in Docker ([Self-Hosting](/docs/bowrain-server/self-hosting))
- **Node.js 18+** (for the example Docusaurus site)

```bash
# Start the server stack (Keycloak + Bowrain)
docker compose up -d --wait

# Verify brain is installed
brain --version
```

## Step 1: Initialize the Project

Navigate to the example Docusaurus project and create a Brain project. You can
create an anonymous project and receive a claim link by email:

```bash
cd examples/docusaurus-e2e

brain init --name "Docusaurus E2E Demo" --email alex@example.com
```

Or create an anonymous project and get the claim URL printed directly:

```bash
brain init --anonymous --name "Docusaurus E2E Demo" --source en
```

This creates a `.brain/` directory with the project configuration and prints a
**claim URL** (or sends it via email) that you'll use in [Step 4](#step-4-claim-the-project):

```
Creating project on https://bowrain.example.com...
Initialized .brain/ project in: /path/to/examples/docusaurus-e2e
Configuration: /path/to/examples/docusaurus-e2e/.brain/config.yaml

Project created: proj_abc123
Claim URL: https://bowrain.example.com/claim/clm_def456

Next steps:
  1. Run: brain push    — upload content to the server
  2. Open the claim URL to take ownership of the project
  3. Invite translators from the web dashboard
```

Use `--json` for machine-readable output in CI/CD pipelines.

## Step 2: Pseudo-Translate for Testing

Before involving real translators, generate pseudo-translations to verify your
i18n setup works. Pseudo-translation replaces characters with accented variants
and adds padding to catch truncation bugs:

```bash
brain pseudo-translate i18n/en/code.json \
  --target-lang qps \
  --output i18n/qps/code.json
```

Preview the pseudo-translated site:

```bash
npm run build -- --locale qps
npm run serve
```

You'll see text like `[!!! Hëëëröööö Tîîîtlëëë !!!]` in the UI — any
truncation or layout issues become immediately visible.

## Step 3: Push to Bowrain

Upload the source content to the Bowrain Server:

```bash
brain push
```

Brain reads the file mappings from `.brain/config.yaml`, parses each file using
the appropriate format reader (JSON, Markdown, etc.), and uploads the
translatable blocks to the server.


## Step 4: Claim the Project

The project was created anonymously on the server. To manage it, **claim** it.
If you used `--email`, check your inbox for the claim link. Otherwise, open the
claim URL printed in [Step 1](#step-1-initialize-the-project).

If you're not signed in, the claim page prompts you to authenticate first. After
signing in, the project is automatically added to your workspace.

The claim flow:

1. Open the claim URL in your browser (or click the link in your email)
2. Sign in with your SSO provider (or create an account)
3. The project is claimed into your personal workspace
4. Click **Go to workspace** to see the project dashboard

:::tip
The claim URL is a one-time-use token. Once claimed, the project belongs to your
workspace and the CLI remembers the association for future `brain push` and
`brain pull` commands.
:::

## Step 5: Invite a Translator

From the Bowrain web app, invite a translator to work on the project:

1. Navigate to **Settings** in the sidebar
2. Go to the **Members** tab
3. Click **Invite member**
4. Enter the translator's email address and select the **Member** role
5. Click **Send invite**

The translator receives an email with an **Accept Invitation** link that takes
them to `/join/\{code\}`, where they can create an account and join the workspace.

```bash
# Or via the API:
curl -X POST http://localhost:8080/api/v1/workspaces/acme/invites \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"email": "translator@example.com", "role": "member", "ttl_days": 7}'
```

## Step 6: Translate in Bowrain

The translator can now work on the project in either the **web app** or the
**desktop app**.

### Web App

After accepting the invite and signing in, the translator sees the project
dashboard. They select a file (e.g., `code.json`) and open the
**Translation Editor**, where they can:

- Translate strings with inline suggestions from Translation Memory
- Look up terminology in the context panel
- Use AI-assisted translation for initial drafts
- Review and approve translations

### Desktop App

If the translator has Bowrain Desktop installed, they'll see an
**Open in Bowrain Desktop** banner on the project view. Clicking it opens the
desktop app via a `bowrain:` deep link (e.g., `bowrain:https://bowrain.example.com/ws/acme/projects/proj_123`),
providing the same translation editor with native performance and offline support.

:::note
The desktop app supports **offline editing**. Translations are synced
automatically when the connection is restored.
:::

## Step 7: Pull Translations

Once translation is complete, pull the finished translations back into your
codebase:

```bash
brain pull --locale nb --locale fr
```

This downloads all translated content and writes it to the mapped file paths:

```
Pulling from http://localhost:8080...
  i18n/nb/code.json                 27 blocks (24 translated)
  i18n/fr/code.json                 27 blocks (22 translated)
  i18n/nb/docusaurus-plugin-content-docs/current.json  4 blocks
  i18n/fr/docusaurus-plugin-content-docs/current.json  4 blocks
Done. 62 blocks pulled across 4 files.
```

Build the translated site to verify:

```bash
npm run build -- --locale nb
npm run serve
```

## Summary

| Step | Tool | What Happens |
|------|------|-------------|
| 1. Initialize | `brain init --anonymous` | Creates project on server, generates claim URL |
| 2. Pseudo-translate | `brain pseudo-translate` | Generate test translations for UI validation |
| 3. Push | `brain push` | Uploads source content to Bowrain Server |
| 4. Claim | Web browser / email | Developer claims the anonymous project |
| 5. Invite | Web app / API | Admin invites translator to workspace |
| 6. Translate | Web / Desktop | Translator works on content |
| 7. Pull | `brain pull` | Downloads translated files into codebase |
