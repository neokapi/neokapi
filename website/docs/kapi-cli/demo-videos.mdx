## Terminology QA

Import a glossary, look up and search terms, then run QA with terminology enforcement.

<video controls autoPlay loop muted width="100%" style={{ maxWidth: "900px" }}>
  <source src="/video/kapi/termbase-qa.webm" type="video/webm" />
</video>

```bash
# Import terms from CSV
kapi termbase import glossary.csv --name product-terms --format csv -s en -t fr

# Check termbase statistics
kapi termbase stats --name product-terms

# Look up a specific term
kapi termbase lookup password --name product-terms -s en -t fr

# Search across all terms
kapi termbase search encrypt --name product-terms -s en

# Run QA with terminology enforcement
kapi qa-check -i translated.json -o qa_report.json \
  --source-lang en --target-lang fr --termbase product-terms
```

## Pre-Translation Pipeline

Full pre-translation workflow: import TM and termbase, leverage TM, translate remaining segments, verify terminology compliance.

<video controls autoPlay loop muted width="100%" style={{ maxWidth: "900px" }}>
  <source src="/video/kapi/termbase-pretranslation.webm" type="video/webm" />
</video>

```bash
# Set up language assets
kapi termbase import glossary.csv --name product-terms --format csv -s en -t fr
kapi tm import project.tmx --name project-tm -s en -t fr

# Step 1: Leverage translation memory
kapi tm-leverage -i messages_en.json -o step1_tm.json \
  --source-lang en --target-lang fr --tm project-tm

# Step 2: Translate remaining segments
kapi pseudo-translate -i step1_tm.json -o step2_translated.json --target-lang fr

# Step 3: Verify terminology compliance
kapi qa-check -i step2_translated.json -o step3_qa.json \
  --source-lang en --target-lang fr --termbase product-terms
```
