# Use Case: Translating a Website

This guide walks through a complete workflow for translating website content using kapi CLI and Bowrain's visual editor.

## Overview

1. **CLI**: Check word count for cost estimation
2. **Bowrain / Web UI**: Create project, translate with visual tools
3. **CLI**: Export translated project as a KAZ archive

## Step 1: Estimate and Open

Use the CLI for estimation and to launch the web editor:

```bash
kapi word-count landing-page.html

# Open a project in the web editor
kapi serve project.kaz
```

### Editor Views

Bowrain offers multiple views for different translation tasks:

| View | Best For |
|------|----------|
| **Grid View** | Overview, batch operations |
| **Split Preview** | Context-sensitive translation with live rendering |
| **Focus View** | Distraction-free, segment-by-segment work |

### Translation Methods

- **Manual**: Type translations directly in the editor
- **AI Translation**: Use configured AI providers for machine translation
- **TM Leverage**: Click "TM Lookup" to auto-fill translations from translation memory
- **Context Panel**: Click "Context" to see per-block TM matches and terminology suggestions
- **Pseudo-Translation**: Generate test translations to find UI issues

### Using Terminology

1. Click **"Terminology"** in the project view to import a termbase (CSV or JSON)
2. In the editor, the **Context panel** shows matched terms with target suggestions for each block
3. Terms with "preferred" status should be used consistently across all translations
4. After translating, run the **term-enforce** flow to validate terminology compliance

## Step 3: Export Translations

After translation, export the project from the content store:

```bash
# Export a project as a KAZ archive
kapi store export <project-id> acme-website.kaz --store gokapi.db
```

## Tips

- **Start with TM Lookup** to auto-fill translations from previous projects
- **Import terminology** before translating to ensure consistent term usage
- **Use the Context panel** for per-block TM and terminology guidance
- **Start with pseudo-translation** to identify potential UI truncation issues
- **Use split preview** when translating content that appears in specific layouts
- **Configure AI providers** in Settings → AI Providers for machine translation
- **Build translation memory** to maintain consistency across projects
