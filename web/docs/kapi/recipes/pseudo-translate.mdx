# Preview before you ship

:::note Runs anywhere
No project needed — pseudo-translation runs ad-hoc on any source catalogue.
:::

**Goal:** see how your interface holds up in another language before a single
real translation exists. kapi fills every string with a longer, accented stand-in,
so the moment your UI re-renders you can spot the labels that clip, the buttons
that overflow, and any text that stayed plain — a sign your code hard-coded it
instead of pulling it from the catalogue. No API key, no waiting, no cost.

## Steps

1. Generate pseudo-translations for your source catalogue.
2. Load the pseudo-translated file into your UI.
3. Look for clipped labels, overflowing buttons, and any string that _didn't_
   get accented — an un-accented string is one your code hard-coded instead of
   pulling from the catalogue.

Pseudo-translation defaults to the **`qps`** pseudo-locale (the BCP-47
private-use locale reserved for pseudo-translation), so you don't need a real
target language.

## Try it

Drop a file (or pick the JSON or Word sample) and kapi pseudo-translates it right
here in your browser — no install, no API key, no server. Every value comes back
accented and wrapped in markers; any string that stays plain is one your code
hard-coded instead of pulling from the catalogue.

## Run it

<Cli>

For scripting and CI it's one command — the output path is a name you choose with
`-o`. Tune the shape with `--prefix`, `--suffix`, and `--expansion-percent` (extra
padding to simulate how much longer real translations tend to run):

```bash
kapi pseudo-translate messages.json -o messages.pseudo.json
```

## In a project

In a `.kapi` project, pseudo-translation runs against the recipe's declared
content and locales, and the output follows the project store instead of an
explicit path.

<Cli>

Run `kapi pseudo-translate` with no file argument, or as a step in a flow you run
with `kapi run`; materialize the result with `kapi merge`.

</Cli>

<Desktop>

Open the project, pick (or compose) the flow, and run it — the pseudo-translated
output lands in the project store, ready to merge out.

</Desktop>

See [Create your first project](/kapi/get-started/first-project).
