# Store AI credentials

**For the concept** of providers and how keys are resolved, see
[Choose a translation provider](/kapi/recipes/choose-a-translation-provider); this
recipe walks the desktop **credential vault**.

**Goal:** save a provider's API key once, securely, so every `translate`, `qa`, or
`rewrite` step that needs it just works — and the key never lands in the committed
recipe.

## Steps

1. **Open the vault.** From **Settings → Providers**, add a credential: pick the
   provider (Anthropic, OpenAI, DeepL, …) and paste its key.
2. **Stored in the keychain.** The key is saved to the OS keychain, never written
   to the `.kapi` recipe — so the recipe stays safe to commit and share.
3. **Use it.** A `translate` step (in the [flow editor](/kapi/desktop/recipes/build-a-flow-visually))
   or any AI-backed tool resolves the credential automatically; no key on any
   command line.
4. **On-device option.** For a free, private alternative, select the **Ollama**
   provider — it runs models locally and needs no key at all.

## Why the desktop

The vault makes credential handling safe by default: keys live in the OS keychain,
the recipe carries only the provider name, and the app surfaces which providers are
configured. The same keychain store backs the CLI's `kapi credentials`, so a key
added here is available to command-line runs too.
