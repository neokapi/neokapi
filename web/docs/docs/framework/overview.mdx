# The neokapi framework

neokapi is a **framework for format-aware localization**. At its core is a
content model that represents the text content independently of the file it
came from, a set of format readers and writers that move documents in and out of
that model, and processing tools — translation, QA, terminology enforcement,
brand-voice checks — that compose into pipelines. It is **open source** and
**offline by default**.

The framework is the thing; everything else is a surface onto it. The same
engine drives the `kapi` command line, the desktop app, and a Go library — to
use it, see [Kapi](/kapi/overview).

## What the framework does

A document — JSON, DOCX, Markdown, HTML, an Android `strings.xml`, an Apple
`.xcstrings`, and [more](/formats) — is read into a **content model**: the text
is extracted into _blocks_, each with a stable id, while the surrounding
structure and inline markup are held separately so a round-trip writes the file
back faithfully. Tools transform blocks as they stream through; a writer
reassembles the original format around the transformed text.

Here is the same file seen two ways — the CLI command on the left, and the
content model the framework parsed it into on the right:

<DualExample
  command="kapi word-count messages.json"

  result={{
    kind: "blocks",
    sample: "messages.json",

The block view is the framework's own representation. Everything else — word
counts, pseudo-translation, AI translation, QA, brand checks — is a tool
operating on these blocks.

## Terminology

If you're familiar with the Okapi Framework, here's how concepts map:

| Okapi (Java)               | neokapi (Go)               |
| -------------------------- | -------------------------- |
| Filter                     | DataFormat (Reader/Writer) |
| Step                       | Tool                       |
| Pipeline                   | Flow                       |
| PipelineDriver             | Executor                   |
| Event                      | Part                       |
| TextUnit                   | Block                      |
| TextFragment               | Run sequence (`[]Run`)     |
| Code                       | Run                        |
| StartSubDocument/SubFilter | Child Layer                |
| Tikal                      | kapi (CLI)                 |
