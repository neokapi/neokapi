Automate translation sync with local rules + GitHub Actions

Generated from walkthroughs/bowrain-automation.md. Do not edit by hand — change the prompt and regenerate.

Bowrain has two layers of automation: local rules in .bowrain/config.yaml that run on the developer machine (e.g. pre-push QA), and GitHub Actions workflows that run on the CI/CD side. Together they make translation sync fully automated — code lands, translations follow.

Scene 1 — local rules​

Inspect .bowrain/config.yaml for local automation rules. Run bowrain push and watch the pre-push QA rule fire automatically before the actual push happens.

Scene 2 — GitHub Actions​

Show the .github/workflows/bowrain-sync.yml that runs bowrain sync on every push to main. This is the CI/CD half of the automation.
