Server Connectors

Connectors integrate Bowrain Server with external systems (CMS, design tools, code repositories, marketing platforms).

Connector Types​

TypeExamplesPurpose
CMSContentful, Sanity, StrapiSource content for translation
DesignFigma, SketchUI text strings from designs
CodeGitHub, GitLab, BitbucketLocalization files in repos
MarketingHubSpot, MarketoCampaign and email content
FileBowrain CLILocal file sync

How Connectors Work​

  1. Pull: Fetch content from external system → Bowrain Server
  2. Process: Translate, review, QA within Bowrain
  3. Push: Send translations back to external system
External System ←→ Connector ←→ Bowrain Server ←→ Translators

File Connector (Bowrain CLI)​

Bowrain CLI acts as a file connector:

# Initialize connection to server
bowrain init --server https://bowrain.example.com --project abc123

# Pull translations from server
bowrain pull

# Push local changes to server
bowrain push -m "Translate new features"

See Bowrain CLI documentation for details.

Server-Side Connectors​

Server-side connectors run on Bowrain Server and integrate with external APIs.

Configuration​

Connectors are configured per workspace in the web UI or via API:

# Create a CMS connector
POST /api/v1/workspaces/:ws/connectors

{
  "type": "contentful",
  "name": "Production CMS",
  "config": {
    "space_id": "abc123",
    "access_token": "...",
    "environment": "master"
  },
  "mappings": [
    {
      "content_type": "blogPost",
      "fields": ["title", "body", "excerpt"],
      "locale_mapping": {
        "en-US": "en",
        "fr-FR": "fr"
      }
    }
  ]
}

Automation​

Connectors can trigger automatic workflows:

# On new content in Contentful
event: connector.content_pushed
source: contentful-prod
action:
  - translate_with_ai
  - run_qa_checks
  - notify_reviewers
