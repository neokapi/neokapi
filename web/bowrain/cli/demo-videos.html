Animated demos of common bowrain CLI workflows. For standalone file-processing demos, see Kapi CLI Demos.

Overview​

Quick tour of bowrain's capabilities.

bowrain --help
bowrain init --help
bowrain status --help

Project Workflow​

End-to-end project workflow: inspect content, count words, and pseudo-translate.

# Initialize a project
bowrain init --name "Landing Page" --source en --targets fr

# Count words for cost estimation
kapi word-count landing-page.html

# Pseudo-translate for UI testing
kapi pseudo-translate landing-page.html --target-lang fr

# Open in web UI for visual editing
bowrain serve

Authentication​

Authenticate the CLI against a deployed bowrain-server using the OAuth device flow.

# Login to a bowrain-server
bowrain auth login --server https://neokapi.example.com

# Check authentication status
bowrain auth status

# Logout
bowrain auth logout

Local Project Server​

Start a web-based translation editor for a local project — no server setup or authentication required.

# Open a project in the browser
bowrain serve

# Custom port, don't auto-open browser
bowrain serve --port 4000 --no-open

Workspace Management​

Create and manage workspaces via the REST API for team collaboration.

# List your workspaces
curl -H "Authorization: Bearer $TOKEN" \
  https://neokapi.example.com/api/v1/workspaces

# Create a team workspace
curl -X POST -H "Authorization: Bearer $TOKEN" \
  -d '{"name": "Acme Translations", "slug": "acme"}' \
  https://neokapi.example.com/api/v1/workspaces

# Create a project in the workspace
curl -X POST -H "Authorization: Bearer $TOKEN" \
  -d '{"name": "Website", "source_locale": "en", "target_locales": ["fr","de"]}' \
  https://neokapi.example.com/api/v1/workspaces/acme/projects

Translation Memory & Terminology​

The bowrain CLI shares TM and termbase commands with kapi. See the Kapi CLI Demos for animated walkthroughs of:

  • Terminology QA — import glossary, look up terms, search, and run QA with terminology enforcement
  • Pre-Translation Pipeline — TM leverage, pseudo-translate remaining segments, verify terminology compliance
# Import terms from CSV
bowrain termbase import glossary.csv --name product-terms --format csv -s en -t fr

# Leverage TM
bowrain tm-leverage -i input.html -o output.html \
  --source-lang en --target-lang fr --tm project-tm

# Run QA with terminology enforcement
bowrain qa-check -i translated.json -o qa_report.json \
  --source-lang en --target-lang fr --termbase product-terms
