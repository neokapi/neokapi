# kapi 1.2.0-rc1

A release candidate covering everything since **1.0.1** — including the work that
shipped in the internal 1.1.0 build, which most users never received. The headline
is the move from a localization-only toolchain to a **format-aware content engine**:
kapi now reads, converts, and faithfully writes back a much wider range of document
formats, and grows a plugin ecosystem for PDF, vision, audio, and video.

## Highlights

### Format-aware content engine
- **Structural conversion** between JSON and YAML that emits a real block-array
  (the `kapi inspect` shape), kept separate from byte-exact catalog round-trips.
- **Cross-format semantic export** — writers own per-format projection of the
  canonical inline model, so links, images, and code-block languages survive a
  trip across formats. New anti-drift coverage guards the round-trips.
- **New formats:** AsciiDoc, and the DocLang structure model for richer
  document geometry.
- **Math:** OMML → MathML/LaTeX conversion, inline and display equation
  surfacing, and `<formula>` LaTeX projection in DocLang.
- **Pretty-printed** HTML semantic export and DocLang writer output.
- **Content fidelity for ingestion** (#928): non-translatable structure,
  captions, comments, and table/list geometry are now surfaced.

### Multimodal
- **Translate extracted content and round-trip across image, audio, and video**,
  with viewing in Kapi Desktop.
- Wider accepted raster formats: GIF, BMP, TIFF, WebP, plus HEIC/AVIF.

### Plugins & model assets
- **Host-owned model assets** — kapi fetches, verifies (pinned sha256), and
  caches plugin models in a shared cache, rather than each plugin self-fetching
  at runtime. Bundled models are listed but not pull/prune-able.
- New plugins available via `kapi plugin install`: **kapi-pdfium** (PDF reader
  with tier-2 structure, glyph geometry, and line grouping), **kapi-vision**
  (OCR / layout), **kapi-asr** (whisper-based speech-to-text), and **kapi-av**
  (bundled ffmpeg audio/video).
- `kapi plugins doctor` for diagnosing plugin installs; bundled native libraries
  are now code-signed.

### CLI
- **`kapi models`** centralizes model management and folds in local-LLM driving
  via **Ollama** (per-block translation with live progress for local providers).
- Consolidated `translate` / QA flows.
- Aligned, `--json`-capable table output across `plugins`, `models`, `tm`, and
  related commands.
- **Self-update**: `kapi update` (cosign-verified) plus a cached update notifier.

### Kapi Desktop
- Content page reorganized into one card per collection, with adaptive content
  workflows and flows surfaced in the toolbox.
- On-demand PDF plugin install, block-store cache fixes, and a sample-upgrade
  prompt. In-app update signing wired in.

### Docs, website & Lab
- Restructured information architecture with a concept-first, source-first
  journey; React removed from the Kapi surface; separate CLI and Desktop sections.
- Interactive in-browser Lab consolidated, with a Conversion Lab, Vision Lab, and
  a self-explanatory format-maturity dashboard.
- Large assets (videos, screenshots, wasm) served from a Cloudflare R2 CDN; real
  React diagram kit replaces ASCII diagrams.

## Install

**Homebrew (kapi CLI, Apache-2.0, no plugins):**
```
brew install neokapi/tap/kapi-cli
```

**Homebrew (kapi + bowrain plugin):**
```
brew install neokapi/tap/bowrain-cli
```

**Homebrew (Kapi desktop app, macOS):**
```
brew install --cask neokapi/tap/kapi
```

**Homebrew (Bowrain desktop app, macOS):**
```
brew install --cask neokapi/tap/bowrain
```

Direct platform binaries (macOS, Linux, Windows) are attached as release assets
below. This is a **pre-release**; install with the `@beta` brew track or download
the assets directly.
