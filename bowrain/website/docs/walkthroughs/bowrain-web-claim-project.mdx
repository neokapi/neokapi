# Claim a project pushed from the CLI

A developer pushes their first project to Bowrain Server using `bowrain push`. Because they haven't created a workspace yet, the server creates the project as **anonymous** and returns a claim URL. The walkthrough shows the developer following that URL, signing in via SSO, and the claim-project flow attaching the project to their newly-claimed workspace.

User opens the claim URL in a browser. They see the "Sign in to claim" landing page (unauthenticated state). After SSO, they land back on the claim page, which fetches workspace info and shows the green "Claim project" CTA. Click claims the project; user is redirected into the workspace dashboard with the new project visible.

The recording covers the full flow: claim URL → sign in → claim → project visible in workspace.
