# Getting started with bowrain — init, push, pull, sync

> Generated from [`walkthroughs/bowrain-getting-started.md`](https://github.com/neokapi/neokapi/blob/main/bowrain/website/walkthroughs/bowrain-getting-started.md). Do not edit by hand — change the prompt and regenerate.

The full Bowrain CLI workflow in four scenes: initialize a project, push source content to the server, pull translated content back, and finally use `bowrain sync` to do all three in one command. This is the canonical "first 5 minutes" experience — by the end, you have a CI-ready workflow.

## Scene 1 — init

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/01-init.webm",
    dark: "/video/bowrain-cli/01-init.webm",
  }}
/>

Run `bowrain init` in your project directory. The command scaffolds `.bowrain/` with config, flow folder, and sync cache. `ls .bowrain/` shows the resulting layout.

## Scene 2 — push

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/02-push.webm",
    dark: "/video/bowrain-cli/02-push.webm",
  }}
/>

`bowrain push` sends your local source content to Bowrain Server. `bowrain status` shows what got pushed and which keys are pending translation.

## Scene 3 — pull

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/03-pull.webm",
    dark: "/video/bowrain-cli/03-pull.webm",
  }}
/>

`bowrain pull` brings translated content back from the server into your local files. `bowrain status` confirms the new state.

## Scene 4 — sync

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/04-sync.webm",
    dark: "/video/bowrain-cli/04-sync.webm",
  }}
/>

`bowrain sync` does push + wait-for-translations + pull in a single command. Use this in CI; the timeout flag bounds how long it'll wait for AI / human translation to complete before exiting.
