# Bowrain Desktop

Bowrain Desktop is a native, cross-platform client for the Bowrain server. It
opens the same workspace as the [web app](/server/web-overview) and is an equal
real-time client — presence, edits, and reviews are shared live across both.
What the desktop adds is **offline-capable editing**: a local cache and an
offline edit queue let you keep working when the network drops, then sync back
when you reconnect.

The desktop app is a **working copy of the server, never a separate source of
truth** — the authoritative content, memory, and configuration live on the
server. Authoring local files and project configuration (the `.kapi` recipe) is
[kapi's job](/getting-started/kapi-vs-bowrain), not the desktop app's.

## The editing surfaces

The desktop uses the same per-file surfaces as the web app — see the
[Translation editor](/server/translation-editor), [Review](/server/review), and
[Pre-process](/server/pre-process) for the full detail:

- **Translate** — edit blocks with AI/TM assistance and terminology insertion,
  in a **Visual** view (an inline card over a formatted document preview) or a
  **Table** view (a source/target grid for scanning and editing many blocks).
- **Review** — work through blocks by status, run QA, and approve or reject
  translations.
- **Pre-process** — file-wide source prep (pseudo-translate, bulk TM leverage)
  before editing begins.

Teammates' presence appears live as they work, and the desktop keeps editing
when offline — changes queue and replay on reconnect (see
[Real-time collaboration](/server/collaboration)).

## Features

- **Three editing surfaces** — Translate (Visual + Table views), Review, and
  Pre-process — sharing one inline editor, inline-tag support, and block-status
  tracking.
- **Real-time collaboration** — presence and live updates shared with the web
  app over the same server.
- **Offline-first** — a local cache and an offline edit queue keep you working
  without a connection; changes replay on reconnect.
- **Per-block context** — TM matches and terminology suggestions with one-click
  apply.
- **Terminology & translation memory** explorers with content-aware tiered
  matching and CSV/JSON/TMX import-export.
- **Flows** — view and edit a project's server-side flows on the shared
  `@neokapi/flow-editor` canvas.
- **AI translation** using Anthropic, OpenAI, Google Gemini, or Ollama, with
  streaming progress for live thinking updates.
- **Remote connectors** — wire in remote content sources (the server owns
  filesystem and git sources; see [Connectors](/server/connectors)).

## Platform

Bowrain Desktop runs as a single native application on macOS, Windows, and
Linux — no additional runtimes or dependencies are needed. Its local store is a
cache and an offline edit queue; the server holds the authoritative content,
translation memory, terminology, and project configuration.
