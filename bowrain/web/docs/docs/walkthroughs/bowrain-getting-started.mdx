# Getting started from a local codebase (kapi)

The kapi developer path: connect a local project to your Bowrain server, push the
source for the team to translate, and pull translations back — synced like git.
This is the on-ramp for a developer working from a local codebase; if your
content lives in a CMS, a design tool, or a git host,
[connect it server-side](/server/connectors) instead — no checkout needed.

`kapi init --server https://bowrain.cloud` writes a `<dir-name>.kapi` recipe and a sibling
`.kapi/` state directory and connects the project to your Bowrain server (pass
`--workspace <slug>` if you belong to several). `kapi add` tracks the files you
want translated; `kapi push` sends the source up for the team and the AI tools
to work on; `kapi status` shows sync state, exactly like git; and `kapi pull`
brings translations back into the same files and formats. `kapi sync` does
push-wait-pull in one command for CI.
