# Getting started from a local codebase (kapi) — init, push, pull, sync

> Generated from [`walkthroughs/bowrain-getting-started.md`](https://github.com/neokapi/neokapi/blob/main/bowrain/web/docs/walkthroughs/bowrain-getting-started.md). Do not edit by hand — change the prompt and regenerate.

The kapi developer path in four scenes: initialize a project, push source content to the server, pull translated content back, and finally use `kapi sync` to do all three in one command — by the end, you have a CI-ready workflow. This is the on-ramp for a developer working from a local codebase; if your content lives in a CMS, a design tool, or a git host, [connect it server-side](/server/connectors) instead — no checkout needed.

## Scene 1 — init

Run `kapi init` in your project directory. The command writes a `<dir-name>.kapi` recipe at the root and creates a sibling `.kapi/` state directory. `ls -la` shows the recipe alongside the state dir; `ls .kapi/` shows the layout (manifest, TM/termbase, `cache/` for regenerable state).

## Scene 2 — push

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/02-push.webm",
    dark: "/video/bowrain-cli/02-push.webm",
  }}
/>

`kapi push` sends your local source content to Bowrain Server. `kapi status` shows what got pushed and which keys are pending translation.

## Scene 3 — pull

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/03-pull.webm",
    dark: "/video/bowrain-cli/03-pull.webm",
  }}
/>

`kapi pull` brings translated content back from the server into your local files. `kapi status` confirms the new state.

## Scene 4 — sync

<ThemedVideo
  sources={{
    light: "/video/bowrain-cli/04-sync.webm",
    dark: "/video/bowrain-cli/04-sync.webm",
  }}
/>

`kapi sync` does push + wait-for-translations + pull in a single command. Use this in CI; the timeout flag bounds how long it'll wait for AI / human translation to complete before exiting.
