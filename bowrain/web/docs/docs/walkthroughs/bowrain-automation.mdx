# Automate translation sync with local rules + GitHub Actions

Bowrain has two layers of automation: **local rules** declared on the `.kapi`
recipe (top-level `hooks:` and `automations:`) that run on the developer machine
(e.g. pre-push QA), and **GitHub Actions workflows** that run on the CI/CD side.
Together they make translation sync fully automated — code lands, translations
follow.

## Local rules

Declare `hooks:` and `automations:` at the top level of the `<dir-name>.kapi`
recipe. A pre-push QA hook, for example, fires automatically before `kapi push`
runs, so source never reaches the server without passing your checks. Local
rules give a developer guardrails on their own machine.

## GitHub Actions

For the CI/CD half, a workflow such as `.github/workflows/bowrain-sync.yml` runs
`kapi sync` on every push to `main`, so the moment code lands the server picks up
the new source, translations are requested, and completed work flows back.
