Acme CloudOps API Reference

Complete reference for the Acme CloudOps REST API. All endpoints require authentication via API key or OAuth 2.0 bearer token.

Authentication

Include your API key in the Authorization header of every request:

Authorization: Bearer YOUR_API_KEY

API keys can be generated from the API Keys page in your account settings. Each key has configurable scopes and expiration.

Projects

List Projects

Returns a paginated list of all projects in your organization.

GET /api/v1/projects

Query Parameters

Parameter Type Description
page integer Page number (default: 1)
per_page integer Items per page (default: 20, max: 100)
status string Filter by status: active, archived, or all

Create Project

Creates a new project in your organization.

POST /api/v1/projects

Request Body

Field Type Required Description
name string Yes Project display name
region string Yes Deployment region (e.g., us-east-1)
plan string No Plan tier: free, pro, or enterprise

Deployments

Create Deployment

Triggers a new deployment for the specified project.

POST /api/v1/projects/:id/deployments

Request Body

Field Type Required Description
branch string No Git branch to deploy (default: main)
environment string No Target environment: staging or production

Rollback Deployment

Rolls back to a previous deployment. This creates a new deployment using the artifact from the specified version. 

POST /api/v1/projects/:id/deployments/:deploymentId/rollback

Error Handling

The API uses standard HTTP status codes. Error responses include a JSON body with a message field describing the error.

Status Code Meaning
400 Bad Request — invalid parameters or request body
401 Unauthorized — missing or invalid API key
403 Forbidden — insufficient permissions
404 Not Found — resource does not exist
429 Too Many Requests — rate limit exceeded
500 Internal Server Error — please contact support

Rate Limits

API requests are limited to 1,000 requests per minute per API key. Rate limit headers are included in every response:

  • X-RateLimit-Limit — Maximum requests per window
  • X-RateLimit-Remaining — Requests remaining in current window
  • X-RateLimit-Reset — Unix timestamp when the window resets
